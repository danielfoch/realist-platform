BUILD A NEW TOOL IN REALIST.CA: “Distress Deals Browser” (DDF MLS)

Goal
Create a separate tool/page on realist.ca that lets users browse MLS listings from the CREA DDF feed that are likely:
1) Foreclosure / Power of Sale / Court-ordered / Bank-owned / Receivership
2) Motivated Seller / Distressed sale language
3) VTB / Vendor Take Back / Seller Financing

This tool should feel like a mini “deal finder” with map + card browse, filters, and a “deal likelihood score” based on MLS description/remarks + listing metadata.

Where it lives
- New route: /tools/distress-deals
- Add to “All Tools” page and main nav under Tools.
- Tool name: “Distress Deals (Foreclosure / Motivated / VTB)”
- Short subtitle: “Find power-of-sale, court-ordered, bank-owned, motivated sellers, and seller-financing opportunities.”

Data source
- Use existing DDF ingestion + database tables used elsewhere in realist.ca (do not re-scrape realtor.ca).
- Query from the same normalized Listings table(s) you already populate from DDF (photos, geo, address, price, beds/baths, property type, remarks/public comments, listing office, listing date, status, etc.).
- Ensure queries are fast: add indexes to remarks text (full-text search where supported) + city + province + status + last_updated + lat/lng.
- Cache query results for 5–15 minutes (server-side) to reduce load.

Core UX
- Layout: left side filters, center map, right side scrollable cards (or map top, cards below on mobile).
- Toggle switches at top:
  [ ] Foreclosure / POS / Court-Ordered
  [ ] Motivated Seller
  [ ] VTB / Seller Financing
  Users can enable one or multiple toggles simultaneously.
- Show a “Distress Score” badge on each card (0–100) with a tooltip: “Score is derived from listing remarks + provincial sale terminology + signals like price reductions and DOM (if available in feed).”
- Clicking a listing opens a detail modal/page with:
  - full remarks (highlight matched terms)
  - matched terms + which category they triggered
  - a short explanation of provincial nuance (based on the listing’s province)
  - contact CTA: “Request the full deal package” / “Talk to an agent” (hook into existing lead capture)

Search/Filter controls
- Province (multi-select), City, Postal prefix, Price range, Beds/Baths, Property type, Status (Active only default)
- Days on market (if available), Listing date range
- Keyword include/exclude (advanced)
- “Include condos / exclude condos” toggle
- “Exclude assignment / new build” toggle if your feed includes such signals (or filter via keywords)

Detection logic
Build a text classifier-like rules engine (not ML required) that:
- Normalizes remarks:
  - lowercase, strip punctuation, collapse whitespace
  - handle common abbreviations (pos, vtb, reo, dom, “court ord.”)
- Searches for terms (word-boundary aware) and assigns points per hit.
- Some terms must be “strong signals” (high points), others “weak signals” (low points).
- Output:
  - categoriesTriggered: {foreclosure_pos: true/false, motivated: true/false, vtb: true/false}
  - matchedTerms: array of {term, category, weight}
  - distressScore: 0–100 (cap, then normalize)
  - confidence: low/med/high based on number of strong terms and term diversity
- Save these outputs on ingestion OR compute on-the-fly with caching. Prefer precompute nightly + incremental recompute on listing updates.

Provincial nuance rules (IMPORTANT)
Because terminology differs:
- Ontario: “Power of Sale” is common; “foreclosure” can appear but POS is the usual mechanism. Treat “power of sale”, “under power of sale”, “mortgagee sale” as strong signals.
- British Columbia: common terms include “court ordered sale”, “foreclosure”, “conduct of sale”, “judicial sale”. Weight those strongly. “Schedule A” / “as is where is” often appears too (medium).
- Alberta: “judicial sale”, “foreclosure”, “court order” common. Weight strongly.
- Quebec (French): include “reprise de finance”, “vente sous contrôle de justice”, “vente judiciaire”, “sous contrôle de justice”, “faillite”, “banque”. Weight strongly; also match “financement vendeur” for seller financing.
- Atlantic provinces: “power of sale”, “foreclosure”, “bank owned” show up; weight moderately but still meaningful.
- Always show a small disclaimer box: “Terminology varies by province; this tool flags likely candidates based on listing language and signals. Confirm with your lawyer/agent.”

Negative filters (reduce false positives)
- If remarks contain “not a power of sale” or “no vtb” etc, subtract points:
  - "no vtb"
  - "vtb not available"
  - "not a power of sale"
  - "not foreclosure"
Also add “Exclude keywords” UI.

Scoring approach (simple + explainable)
- Each matched STRONG term: +20
- Each matched MEDIUM term: +8
- Pattern match combos: +12
- Cap raw score at 120 then normalize to 0–100.
- Confidence:
  - HIGH if >=2 strong hits OR strong hit + 2 medium from same category
  - MED if raw score >=25
  - LOW otherwise
- Store matched terms so you can highlight them in UI.
