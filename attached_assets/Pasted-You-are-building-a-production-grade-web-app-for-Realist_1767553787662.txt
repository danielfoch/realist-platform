- Auth: optional, but at minimum support “anonymous lead + saved analyses linked to lead”
- Charts: Recharts
- Form validation: Zod + React Hook Form
- State: React Query (TanStack Query)
- PDF/Export: html-to-image for screenshots + jsPDF for PDF export
- Maps: Google Maps Places API (autocomplete + address validation + geocoding)

CRM INTEGRATION (HARD REQUIREMENT)
Must make it possible to automatically add leads to GoHighLevel/HighLevel (GHL).
Implement:

- On lead creation, POST JSON payload with lead fields + analysis metadata
- Retry logic with exponential backoff
- Store webhook delivery logs in DB (success/fail/response)
- Provide admin screen showing recent deliveries
NOTE: Do NOT hardcode GHL endpoints; use a generic webhook pattern compatible with HighLevel workflows.

DEAL LOCATION (HARD REQUIREMENT)
Require exact deal address using Google Places Autocomplete.
- User types address -> choose suggestion
- Store: formatted_address, place_id, lat/lng, city, region/state, country, postal/zip
- Use country to determine logic: Canada vs USA

COUNTRY LOGIC
If country = USA:
- Provide a “US Mode” deal calculator similar to BiggerPockets rental calculator, but “better and sexier”
- Include rental assumptions, financing, rehab, operating expenses, projections, IRR
If country = Canada:
- Provide strategy options:
  1) Buy & Hold (must include CMHC MLI Select support + points for retrofits)
  2) Flip
  3) BRRR
  4) Airbnb (short-term rental)
  5) Land Assembly
  6) Multiplex
For Multiplex:
- Ask unit count
- If units <= 4: conventional financing path
- If units >= 5: CMHC MLI Select financing path
- Support construction financing:
  - Conventional construction financing
  - CMHC MLI construction financing
  - Conventional construction with CMHC MLI takeout
- Must compute equity-out in BRRR and CMHC takeout scenarios

DEFAULT CONSERVATIVE ASSUMPTIONS (CANADA)
- Vacancy default: 5%
- Rent growth default: 0%
- Allow user to adjust, but conservative defaults are preset

DATA SOURCES (IMPLEMENT WITH SAFE FALLBACKS)
We need:
- Debt rates (approx) pulled regularly from public sources (WOWA, Ratehub, etc.)
- CMHC MLI Select: compute rate using Canada 5-year bond yield + spread
- CMHC market vacancy and rent growth (by region) – start with fallback if API not available

Implement an internal “Rates & Market Data” service with:
- Daily cache table: interest_rate_cache, bond_yield_cache, cmhc_market_cache
- Admin button: “Refresh data now”
- Scheduled refresh (cron-like) is optional if Replit supports; otherwise do refresh on first request each day
- Use safe fallback values if fetch fails (clearly label as “estimates”)
IMPORTANT: Do not rely on scraping that violates terms. Prefer APIs, RSS, or manually-maintained JSON endpoints. If real-time sources are not accessible, implement the architecture and use placeholder endpoints with instructions.

CALCULATIONS (CORE)
Compute headline metrics for all strategies:
- Cap Rate (NOI / purchase price)
- Cash-on-Cash Return (annual pre-tax cash flow / cash invested)
- DSCR (NOI / debt service)
- IRR (levered, multi-year horizon; default 10 years, adjustable)
Also compute:
- Monthly cash flow
- NOI
- Total cash invested
- Equity growth over time
- Principal paydown
- Refinance proceeds where applicable (BRRR/takeout)

MODEL REQUIREMENTS
- Build a robust financial engine with clear separation:
  - inputs schema (Zod)
  - strategy calculators (per strategy)
  - shared utilities (amortization, IRR, NPV)
- IRR implementation must be reliable and tested (use iterative method with bounds, handle non-convergence gracefully)
- Use monthly projection model:
  - revenues (rent/airbnb)
  - expenses (fixed + variable)
  - financing
  - capex/rehab schedule
  - refi/takeout event timing (BRRR)
  - sale event at end (with selling costs)
- Add unit tests for amortization + IRR + DSCR

STRATEGY INPUTS (DETAILS)
Common inputs:
- Purchase price
- Closing costs
- Down payment %
- Financing type + rate + amortization + term
- Gross rents (unit-level optional for multiplex)
- Vacancy %
- Operating expenses:
  - property tax
  - insurance
  - utilities (water/hydro/gas)
  - maintenance %
  - management %
  - repairs/capex reserve %
  - snow/landscaping (Canada-friendly)
- Rent growth %
- Expense inflation %
- Sale assumptions: exit cap (or appreciation %), selling costs %

Flip:
- Purchase, rehab budget, holding costs, ARV, timeline months
- Lending: private/hard money option + fees
- Output: projected profit, ROI, annualized ROI, sensitivity to ARV and time

BRRR:
- Rehab budget and schedule
- After-repair value (ARV)
- Refi LTV, refi rate/term
- Cash-out amount
- Output: “Infinite return?” flag + cash left in deal + stabilized metrics

Airbnb:
- ADR, occupancy %, seasonality toggle (simple)
- Cleaning fees and platform fees
- Furnishing upfront costs
- STR management option
- Output: monthly revenue, net cash flow, sensitivity

Land Assembly:
- Multiple parcels (simple list)
- Carrying costs
- Time-to-entitlement
- Probability-weighted outcomes (base case)
- Output: expected value + risk notes (keep simple v1)

Buy & Hold + CMHC MLI Select (Canada):
- MLI Select points estimator for retrofits:
  - Provide a points input UI (checkboxes for energy efficiency retrofits)
  - Store total points; reflect in premium reductions or better rates (approx modeling)
- Use bond yield + spread for rate; allow user override

Multiplex + Construction + MLI Takeout:
- Construction budget, draw schedule
- Interest-only during construction
- Takeout date and takeout LTV
- Equity extracted at takeout
- Output: total project cost, stabilized value, refi proceeds, remaining equity
