# Platform Constraints

## Existing conventions to extend

1. PostgreSQL/Drizzle is the system of record; IDs are UUID-like `varchar` values generated by `gen_random_uuid()`.
2. `shared/schema.ts` already declares a PostGIS geometry passthrough; spatial work belongs in Postgres/PostGIS rather than a second GIS store.
3. Server jobs use `node-cron`, explicit timezone-aware dates, exported run functions, and database audit/state tables.
4. React report pages use cards, badges, Recharts, responsive layouts, stable test IDs, and source/date footers.
5. `MultiplexUnderwriterPage.tsx` already labels figures `verified`, `inferred`, `assumption`, or `estimate` and compares MLI hold with condo termination.
6. `MarketReportBuilder.tsx` provides the saved-report and chart composition pattern.
7. API routes are Express, typed at the boundary, authenticated where appropriate, and return explicit success/error envelopes.
